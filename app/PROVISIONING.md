## Set each value

### `FORGE_SECRETS_KEY` — Required

- **Capability:** C5 · Secrets vault (master key)
- **What:** Master key the data-plane sidecar uses to DECRYPT the app’s C5 secrets vault at rest.
- **When needed:** Required whenever the app uses any secret or the agent runtime (C1). Use the SAME key the secrets were sealed under — rotating it orphans the vault.
- **Obtain:** Generate once, store securely, reuse across deploys.
- **Generate:** `openssl rand -base64 32`
- **Dev / control plane** (encrypted C5 vault): `./forge secrets set --app forge-os --name FORGE_SECRETS_KEY --from-env FORGE_SECRETS_KEY`
  (export `FORGE_SECRETS_KEY` first, or use `--value <v>`; `--from-env` keeps it out of shell history).
- **Prod / deployed**: put `FORGE_SECRETS_KEY=<value>` in `app/.env.prod`, then redeploy so it's injected: `./forge deploy --app forge-os`.
